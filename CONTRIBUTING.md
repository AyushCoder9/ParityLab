# Contributing

1. Read `AGENTS.md` and `docs/STATE.md`.
2. Keep changes inside one workstream when possible.
3. Add tests for every behavior change.
4. Run `make verify` before requesting review.
5. Explain the customer effect, failure modes, and evidence in the pull request.

Commits should be small, imperative, and independently testable. Never commit secrets or generated reports containing user data.
